# Password hashing

crypto-hasher includes memory-hard and adaptive password hashing suitable for storing credentials. Prefer **Argon2id** for new applications unless you have a specific reason to use bcrypt or scrypt.

## Argon2

Implements Argon2i, Argon2d, and Argon2id (version 1.3 / `v=19`).

| Variant | Use when |
| ------- | -------- |
| `argon2id` | Default choice for password hashing (hybrid) |
| `argon2i` | Side-channel resistance is the priority |
| `argon2d` | GPU/ASIC resistance is the priority (not recommended for passwords alone) |

### Hash and verify

```js
import { argon2id, argon2Verify } from "crypto-hasher";

async function run() {
  const salt = crypto.getRandomValues(new Uint8Array(16));

  const hash = await argon2id({
    password: "pass",
    salt,
    parallelism: 1,
    iterations: 256,
    memorySize: 512, // KiB
    hashLength: 32,
    outputType: "encoded",
  });

  const isValid = await argon2Verify({
    password: "pass",
    hash,
  });

  console.log(isValid ? "Valid" : "Invalid");
}
```

### Options

| Option | Description |
| ------ | ----------- |
| `password` | Password or message (`IDataType`) |
| `salt` | Random salt, **at least 8 bytes** |
| `secret` | Optional keyed hashing secret |
| `iterations` | Time cost (≥ 1) |
| `parallelism` | Parallel lanes (≥ 1) |
| `memorySize` | Memory in **kibibytes**; must be ≥ `8 * parallelism` |
| `hashLength` | Output length in bytes (≥ 4) |
| `outputType` | `'hex'` (default), `'binary'`, or `'encoded'` |

Use `outputType: "encoded"` when you plan to call `argon2Verify`. The encoded string embeds the algorithm, parameters, salt, and hash.

Parameter selection guidance: [IETF Argon2 draft §4](https://tools.ietf.org/html/draft-irtf-cfrg-argon2-04#section-4).

## bcrypt

```js
import { bcrypt, bcryptVerify } from "crypto-hasher";

async function run() {
  const salt = crypto.getRandomValues(new Uint8Array(16)); // must be 16 bytes

  const hash = await bcrypt({
    password: "pass",
    salt,
    costFactor: 11, // 4–31
    outputType: "encoded",
  });

  const isValid = await bcryptVerify({ password: "pass", hash });
}
```

Constraints:

- Password length: 1–72 bytes
- Salt length: exactly 16 bytes
- `costFactor`: integer from 4 to 31
- Default `outputType` is `'encoded'`

## scrypt

```js
import { scrypt } from "crypto-hasher";

const key = await scrypt({
  password: "password",
  salt: crypto.getRandomValues(new Uint8Array(16)),
  costFactor: 1024, // power of 2, ≥ 2
  blockSize: 8,
  parallelism: 1,
  hashLength: 32,
  outputType: "hex",
});
```

| Option | Description |
| ------ | ----------- |
| `costFactor` | CPU/memory cost; must be a power of 2 ≥ 2 |
| `blockSize` | Block size parameter (8 is common) |
| `parallelism` | Parallelization factor |
| `hashLength` | Output size in bytes |
| `outputType` | `'hex'` (default) or `'binary'` |

scrypt has no built-in verify helper — compare derived keys yourself using a constant-time comparison when verifying.

## Choosing an algorithm

| Need | Recommendation |
| ---- | -------------- |
| New password storage | **Argon2id** with `encoded` output |
| Legacy bcrypt hashes | `bcrypt` / `bcryptVerify` |
| Interop with scrypt-based systems | `scrypt` |
| Key stretching for encryption keys | Prefer Argon2id or [PBKDF2](/guides/pbkdf2) per your protocol |
