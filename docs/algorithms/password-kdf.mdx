# Password hashing & key derivation

| API | Role |
| --- | ---- |
| `argon2i` / `argon2d` / `argon2id` + `argon2Verify` | Modern password hashing (preferred: Argon2id) |
| `bcrypt` + `bcryptVerify` | Legacy-compatible password hashing |
| `scrypt` | Memory-hard KDF |
| `pbkdf2` | Iterated HMAC-based KDF |
| `createHMAC` | Message authentication over any digest |

## Minimal Argon2id example

```js
import { argon2id, argon2Verify } from "crypto-hasher";

const salt = crypto.getRandomValues(new Uint8Array(16));

const hash = await argon2id({
  password: "secret",
  salt,
  iterations: 256,
  parallelism: 1,
  memorySize: 512,
  hashLength: 32,
  outputType: "encoded",
});

await argon2Verify({ password: "secret", hash }); // true
```

Always generate a fresh random salt per password. Store the `encoded` string (or salt + parameters + hash) so verification can reconstruct the inputs.
